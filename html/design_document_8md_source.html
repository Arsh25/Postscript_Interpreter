15 ##Goal
16 Produce a C++ library that takes commands to draw certain shapes and outputs code in the PostScript programming language.
17 
18 ##Testing Framework
19 We are using the [Catch](https://github.com/philsquared/Catch) framework for our testing. This is an easy to use testing framework that only needs one header and one #include.
20 
21 We are also utilizing [Travis CI](https://travis-ci.org/) to automate the testing. When new code is pushed onto GitHub, Travis CI is called to build the testing code and report any errors.
22 
23 ##Documentation
24 We are utilizing a documentation generation tool from source code called [Doxygen](http://www.stack.nl/~dimitri/doxygen/). It generates an online documentation browser (HTML) which makes it easy to navigate the source code. It also makes it easy to see the inheritance and dependencies because of the automatic generation of graphs and diagrams.
